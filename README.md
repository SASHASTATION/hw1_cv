# README

Стабилизация видео на основе оптического потока и глобальной модели движения камеры.

**Вариант A** 
1. Выбрать видеоролик с заметной дрожью камеры (телефонная съёмка, action-камера, GoPro, записанная вручную панорама).

2. Вычислить плотный оптический поток (Farnebäck) или разреженный (LK) и оценить глобальное движение камеры по кадрам (аффинная модель или гомография).

3. Сгладить траекторию движения низкочастотным фильтром (скользящее окно, экспоненциальное сглаживание или фильтр Калмана по желанию).

4. Выполнить **компенсацию движения** через warp каждого кадра к стабилизированной траектории.

5. Сформировать стабилизированное видео.

6. Построить визуальные сравнения «до/после» и провести **error analysis**: какие участки стабилизируются плохо и почему (оптический поток, тени, motion blur, нехватка текстуры, ошибки глобальной модели)
  

---

  

## Что делает проект

  

1. Вычисляет **плотный оптический поток** (Farnebäck) между соседними кадрами.

2. Сэмплирует поток на регулярной сетке и оценивает **глобальное движение камеры** — на выбор:

- **Аффинная модель** 

- **Гомография** 

3. Накапливает **траекторию камеры** (x, y, angle) и сглаживает её одним из трёх фильтров:

- **Фильтр Калмана** 

- **Скользящее среднее** 

- **Экспоненциальное сглаживание** 

4. Выполняет **warp** каждого кадра к сглаженной траектории и записывает стабилизированное видео.

5. Строит визуализации и проводит **error analysis**.

## Структура проекта

  

```

├── notebooks/

│ └── analysis_affine.ipynb # error analysis 
│ └── analysis_homography.ipynb # error analysis 

├── input/ # входные видео 

└── output/ # результаты: видео, графики, кадры

```

## Быстрый старт

  

### Установка

  

```bash

pip install -r requirements.txt

```

  

### Запуск

  

```bash

# нужно положить видео в input/video.mp4 (или указать  путь через --input, но в гите есть видео для примера )

  

# Аффинная модель + Калман (по умолчанию)

python run.py

  

# Гомография + скользящее среднее

python run.py --model homography --smoother moving_avg

  

# С указанием входного файла и кадров для сравнения

python run.py --input input/shaky_gopro.mp4 --save-frames 0 50 100 200 400

```

  

### Параметры CLI

  

| Параметр | По умолчанию | Описание |

|----------|-------------|----------|

| `--input` | `input/video.mp4` | Путь к входному видео |

| `--output-dir` | `output` | Папка для результатов |

| `--model` | `affine` | `affine` или `homography` |

| `--smoother` | `kalman` | `kalman`, `moving_avg` или `exponential` |

| `--crop-scale` | `1.04` | Масштаб обрезки чёрных краёв |

| `--grid-step` | `15` | Шаг сетки для потока |

| `--save-frames` | `0 50 100 200` | Индексы кадров для до/после |

## Описание  работы

В работе выполнены все этапы задания варианта A: вычисление оптического потока, оценка глобальной модели, сглаживание траектории, компенсация движения и формирование стабилизированного видео.

Реализованы два ноутбука — для аффинной модели и гомографии — чтобы количественно и визуально сравнить оба подхода. Оба ноутбука имеют одинаковую структуру, но содержат немного разные выводы, основанные на полученных метриках.

На основе метрик и визуального восприятия лучше использовать гомографию, однако в связи с тем, что данные подходы используют идейно один и тот же подход качество финального вывода у них похоже. 

Также в своей работе я помимо  стабилизированных видео создал  диагностические видео с визуализацией стрелок оптического потока, стрелок глобальной модели и метрик покадрово — для наглядной демонстрации выводов error analysis.

В github для удобства запушил весь контекс с input и output.
